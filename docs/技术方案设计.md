## Context（背景与目标）

`AI佣金查询工具需求书.md` 原始设计基于 **Python(FastAPI) + Vue.js + LangGraph(Python) + Pydantic** 技术栈，
为保险经纪人（Broker）提供"对话流 + 结构化卡片"的自然语言佣金测算服务。本期（Phase 1）只做**沙盒模拟测算台**，
不连真实保单系统。

用户要求：**后端改用 TypeScript 技术栈，前端先做网页、后续 app + 小程序。** 本方案把需求书里的每一处
Python/Pydantic/FastAPI/LangGraph 设计，等价翻译为 TypeScript 生态的实现，并固化以下技术选型（已与用户确认）：

| 维度 | 选型 | 说明 |
|---|---|---|
| 前端 | **Flutter** | 一套 Dart 代码 → Web/H5（本期）、iOS/Android App（后续）；小程序 Phase 2 用薄渲染器复用同一契约 |
| 后端 | **NestJS** | 模块化 + DI + 装饰器，最接近 FastAPI 的结构，适合 Admin + HITL + 多引擎 |

**关键不变量（来自需求书，必须保留）**：
- 前端零业务逻辑，只按 `type` 字段做多态渲染（Server-Driven UI）。
- `Ask_User_Node` 每次**只反问一个**缺失槽位（单一反问红线）。
- 每个对话绑定 `thread_id`，断线可精准恢复到中断的反问节点。
- AI 生成的计算代码**严禁主进程 `eval`**，必须在沙箱执行（3s/128MB/断网/只读）。
- 配置版本**只追加不物理删除**，旧版本 `effective_end_date` 日落，可追溯历史费率。

## 共享契约层 `packages/contracts`（Zod，替代 Pydantic）

这是全系统的"单一事实源"，前后端 import 同一份。对应需求书 3.3 / 3.4 / 第 6 章。

```ts
// render-command.ts —— UIRenderCommand 判别联合 (对应第6章 Pydantic Union)
import { z } from 'zod';

export const OptionItem = z.object({ label: z.string(), value: z.string() });

export const ContextTag = z.object({           // 卡片顶部可点击微调标签
  label: z.string(), slotKey: z.string(), value: z.string(),
  editable: z.boolean().default(true),
});

export const RiskBanner = z.object({
  level: z.enum(['info', 'warning', 'danger']),
  text: z.string(),
});

export const TextCommand = z.object({ type: z.literal('text'), content: z.string() });

export const ChoiceChipsCommand = z.object({
  type: z.literal('choice_chips'), content: z.string(),
  options: z.array(OptionItem), targetSlot: z.string(),
});

export const NativeInputCommand = z.object({
  type: z.literal('native_input'), content: z.string(),
  inputType: z.enum(['text', 'number']),          // 控制原生键盘 (数字键盘)
  placeholder: z.string().nullish(), targetSlot: z.string(),
});

export const AccordionRow = z.object({
  year: z.string(), basic: z.string(), allowance: z.string(),
  total: z.string(), tooltip: z.string().nullish(),
});

export const CommissionCardCommand = z.object({
  type: z.literal('commission_card'),
  traceId: z.string(),                            // 点踩/报错溯源
  header: z.string(), heroNumber: z.string(),
  contextTags: z.array(ContextTag),
  accordionData: z.array(AccordionRow),
  riskBanner: RiskBanner.nullish(),
});

export const ComparisonCardCommand = z.object({   // 双产品对比
  type: z.literal('comparison_card'), traceId: z.string(), header: z.string(),
  contextTags: z.array(ContextTag),
  comparisonData: z.array(z.object({
    metricName: z.string(), productAValue: z.string(),
    productBValue: z.string(), highlight: z.enum(['A', 'B', 'None']),
  })),
  riskBanner: RiskBanner.nullish(),
});

export const UIRenderCommand = z.discriminatedUnion('type', [
  TextCommand, ChoiceChipsCommand, NativeInputCommand,
  CommissionCardCommand, ComparisonCardCommand,

export type UIRenderCommand = z.infer<typeof UIRenderCommand>;
```

## AI 编排层详设（LangGraph.js）

对应需求书 3.2。LangGraph.js 与 Python 版 API 几乎一一对应：`StateGraph` + `Annotation` 定义状态、
`addConditionalEdges` 做路由、`interrupt()` 做反问挂起、`PostgresSaver` 做 checkpoint。

```ts
// orchestrator/state.ts —— 对应 3.3 BrokerDialogState
import { Annotation, messagesStateReducer } from '@langchain/langgraph';

export const DialogState = Annotation.Root({
  threadId: Annotation<string>(),
  messages: Annotation<BaseMessage[]>({ reducer: messagesStateReducer, default: () => [] }),
  // 核心槽位
  company: Annotation<string | null>(),
  productName: Annotation<string | null>(),
  premiumTerm: Annotation<string | null>(),       // '5' | '10' | 'Single'
  investorStatus: Annotation<'PI' | 'Non-PI' | null>(),
  premiumAmount: Annotation<number>({ default: () => 100000 }),
  policyYear: Annotation<number | null>(),
  // 状态机控制
  missingSlots: Annotation<string[]>({ default: () => [] }),
  uiRenderCommands: Annotation<UIRenderCommand[]>({ default: () => [] }),
});
```

**5 个节点（与需求书 3.2 完全对齐）**：

1. `extractorNode` — 调 `LlmProvider` 做意图/实体提取，结合 `messages` 历史，覆盖写入 State 槽位。
   - `messageType === 'slot_update'` 时**跳过 LLM**，直接赋值 `state[targetSlot] = slotValue`（省 Token，对应 6.2 设计意图）。
2. `routerCondition`（条件路由，决策枢纽）— 校验必填槽位，计算 `missingSlots`。
   - `missingSlots` 非空 → `ask_user`；已清空 → `compute`。
3. `askUserNode`（动态反问）— **只取 `missingSlots[0]`** 生成反问，按槽位类型决定下发
   `ChoiceChipsCommand`（枚举：年期/身份）或 `NativeInputCommand`（数值：保费）。**用 `interrupt()` 挂起**，
   等前端补齐后从 checkpoint 恢复。
4. `computeNode`（沙盒引擎计算）— 槽位齐备后调 `engineRegistry.resolve(company).calculate(state)` 执行无状态测算。
5. `synthesisNode`（结果拼装 + 风险拦截）— 格式化为 `CommissionCardCommand`，并跑 `RiskInterceptor`
   比对预警规则库，命中则强制写入 `riskBanner`（如大额退佣预警）。

```ts
// orchestrator/graph.ts (骨架)
const graph = new StateGraph(DialogState)
  .addNode('extractor', extractorNode)
  .addNode('ask_user', askUserNode)
  .addNode('compute', computeNode)
  .addNode('synthesis', synthesisNode)
  .addEdge(START, 'extractor')
  .addConditionalEdges('extractor', routerCondition, { ask_user: 'ask_user', compute: 'compute' })
  .addEdge('compute', 'synthesis')
  .addEdge('synthesis', END)
  .addEdge('ask_user', END)                       // interrupt 后本轮结束, 等用户回包
  .compile({ checkpointer: postgresSaver });      // thread_id 持久化

// 调用: graph.invoke(input, { configurable: { thread_id } })
```

**会话失效（3.1）**：`thread_id` 失效/JWT 过期 → 返回 `SessionExpired` 状态码；前端不弹刺眼报错，
清屏重渲染欢迎语。

## 计算引擎层（对应需求书第 4 章）

**统一入参、独立引擎**：抽象基类 + 每保司一个子类，保司差异（万通高龄递减、宏利 Band）只在自身子类实现。

```ts
// engines/base.engine.ts
export abstract class BaseCommissionEngine {
  abstract readonly company: string;
  abstract calculate(state: CalcInput): CommissionResult;   // 无状态纯函数
  protected abstract loadConfig(key: ConfigKey): RatesByYear; // 从 Redis 预热缓存取
}

// engines/registry.ts —— Phase 5 热重载 / 一键回滚的挂载点
class EngineRegistry {
  private engines = new Map<string, BaseCommissionEngine>();
  register(e: BaseCommissionEngine) { this.engines.set(e.company, e); }
  resolve(company: string): BaseCommissionEngine { /* ...含 fallback */ }
}
```

**永明 SunLifeEngine 数学契约（4.2 节，逐字实现）**，变量 `N` = Policy Year：
- 步骤一 基础佣金：`basicComm_N = premiumAmount * config.basicRate_N`（基数永远是保费）。
- 步骤二 津贴资格：`N ∈ eligibleYears` 才发，否则所有津贴 = 0（PI 通常仅首年，Non-PI 顺延续期）。
- 步骤三 额外津贴（基数是**当年基础佣金**，非保费）：
  `extra = basicComm_N * extraRate`，`smpa = basicComm_N * smpaRate`，`ma = basicComm_N * maRate`。
- 步骤四 聚合：`totalComm_N = basicComm_N + extra + smpa + ma`。
- 入库时第 2 年起若超出 eligibleYears，JSONB 里 allowances 已被导入脚本置 0 → 引擎统一做加法，**无需 if-else 判空**（4.1 附注）。

**Risk Interceptor 退佣拦截（4.2(4)）**：
- 全局：`policyYear === 1` 追加"保费退回/欺诈则全额退回 Extra/SMPA/MA"。
- 特定：`productName` 含 `Bright Universal Life` 且 `policyYear===1` 且 `premiumAmount > 300000`
  → 强制红色 `danger` banner（首 12 个月提取 ≥ 50% 触发 Claw Back SMPA）。

费率/金额阈值**全部抽离到配置表**，引擎只读 Schema（逻辑与费率解耦，对应 1.2）。

## 数据库设计（PostgreSQL + JSONB，Drizzle）

对应需求书 4.2(2) / 5.1 / 5.4。核心是**高内聚单一配置表 + JSONB 载体 + 版本日落**。

```ts
// config/schema.ts (Drizzle)
export const commissionConfig = pgTable('sunlife_commission_config', {
  id: serial('id').primaryKey(),
  productName: text('product_name').notNull(),
  premiumTerm: text('premium_term').notNull(),
  investorStatus: text('investor_status').notNull(),
  effectiveDate: date('effective_date').notNull(),
  effectiveEndDate: date('effective_end_date'),           // 日落: 旧版本设为新版前一天
  status: text('status').$type<'Draft' | 'Active' | 'Sunset'>().notNull(),
  ratesData: jsonb('rates_data').$type<RatesByYear>().notNull(),  // O(1) 取全年费率
}, (t) => ({
  uniq: uniqueIndex().on(t.productName, t.premiumTerm, t.investorStatus, t.effectiveDate),
}));
```

`ratesData` JSONB 契约（4.2(2)）：
```json
{ "rates_by_year": {
    "1": { "basic_rate": 0.25, "allowances": { "extra": 0.60, "smpa": 0.35, "ma": 0.18 } },
    "6": { "basic_rate": 0.00, "allowances": { "extra": 0, "smpa": 0, "ma": 0 } } } }
```
其余表：`unhandled_queries`（未命中池）、`feedback`（点踩 HITL）、`broker_whitelist`（手机号白名单）、
`engine_versions`（AI 生成代码版本，供回滚）。**严禁物理 DELETE 任何配置**，全部靠 status + 日期轴追溯。

## 配置导入管道 + Admin + 零信任沙箱 + HITL（需求书第 5 章）

**Excel 导入管道（5.4，`exceljs` 替代 Pandas）**：
1. 文件名正则提取 `effective_date`（`SunLife_Rates_20260101生效.xlsx` → `2026-01-01`）。
2. **逆向展平**：`Yr 3-5: 0.027` 自动展开为 `{ '3':.027, '4':.027, '5':.027 }`。
3. **Zod 熔断校验**：费率非小数（"20%"/"百分之二十"）、年度越界（>100）→ 立即熔断，返回**具体 Sheet+行列号**打回。
4. 两 Sheet（基础佣金 + 津贴周期）按 `Product+Term` Join → 聚合 JSONB → 写库标 `Draft`。

**零信任沙箱（5.2，`isolated-vm`）**——AI Coder Agent 现在生成 **TS/JS** 而非 Python：
- 内存上限 `128MB`（isolate `memoryLimit`）、单次超时 `3000ms`（`script.run({ timeout })`）。
- 隔离 V8 isolate **天然无 Node/网络/文件系统访问**（不注入 `require`/`fetch`/`fs`），满足断网+只读+禁高危库。
- 主进程**严禁 `eval/Function`**，只通过 isolated-vm 传值调用。可叠加 `worker_thread` 做二次隔离。

**HITL 5 步安全闭环（5.3 + 5.4 Step4）**：
1. 未命中拦截 → 写 `unhandled_queries`，Admin 红点。
2. 运营传费率书 → Coder Agent 生成 TS 计算类。
3. 沙箱自动跑数十组边界用例（保费=0/9999999、年龄=100、非法枚举）；崩溃/算出负数 → Agent 读 stack 自我重构，**最多 3 次**。
4. **人工放行**（关键拦截点）：全绿不自动上线，Admin 双栏看《测试报告》+ 代码片段，手点 `Approve & Deploy`。
5. **热重载 + 一键回滚**：`EngineRegistry` 动态挂载新引擎（无需重启）；出问题 `1-Click Rollback` 切回上一稳定版本 Tag。

**Admin 可视化（5.1）**：JSONB 配置"反向解析"渲染成 Excel 样的保单年度矩阵表供运营审查。

## 前端设计（Flutter, Server-Driven UI）

对应需求书第 2 章 + 6.1。前端是**纯渲染器**；后端 Zod 契约是协议源头，Flutter 侧用 `lib/models/` 的
Dart 类镜像同一套 JSON（`UiCommand.fromJson` 按 `type` 解析, 未知类型降级为 `UnknownCommand`）。核心是按
sealed 类型穷尽多态分发：

```dart
// lib/widgets/command_renderer.dart
Widget build(BuildContext context) => switch (command) {
  TextCommand c           => TextBubble(content: c.content),
  ChoiceChipsCommand c    => ChoiceChipsWidget(command: c, active: active, onSelect: onSlot),
  NativeInputCommand c    => NativeInputWidget(command: c, active: active, onSubmit: onSlot),
  CommissionCardCommand c => CommissionCardWidget(command: c, /* onEditTag/onFeedback */),
  ComparisonCardCommand c => ComparisonCardWidget(command: c),
  UnknownCommand()        => const FallbackBubble(),   // 6.1 未知指令降级, 不白屏
};
```

**关键交互（第 2 章）**：
- **UI Lock 防并发**：发消息/点选项后立即置灰所有可交互元素 + 骨架屏，收到新状态才解锁（阻断竞态）。
- **Choice Chips**（枚举：年期/身份）：横向滚动卡片，点击即发 `{messageType:'slot_update', targetSlot, slotValue}`，屏蔽键盘。
- **Native Input**（数值：保费/年龄）：聚焦输入框 + `inputType:'number'` 唤起原生数字键盘。
- **佣金卡片 5 分区**：Header / 可点击 ContextTag 微调区 / Hero 核心数字 / 折叠明细表(Accordion) / 风险 banner。
  - 点 ContextTag 弹窗改值 → 触发 `slot_update` 局部重算（无需重新对话）。
- **NLP 静默微调**：输入框直接打字"如果换 10 年期呢" → `messageType:'text'`，后端覆盖 State 1 秒内出新卡片。
- **HITL 点踩**：卡片右下角"报错"按钮 → 弹反馈框 → `messageType:'feedback' + traceId` 推未命中池。
- **AI 人设（2.5）**：三不要（不寒暄/不长篇/不脑补）写进 System Prompt，反问极简（"请问该保单的缴费年期是？"）。
- **隐私拦截（7.3）**：输入框上方常驻"⚠️ 内部沙盒工具，勿输入客户真实姓名/HKID"。
- **跨端**：本期 Flutter→Web/H5；Phase 2 同 Dart 代码→iOS/Android App；小程序用独立薄渲染器复用契约，后端契约不变。

## LLM 适配层（国内模型）

```ts
// llm/provider.ts —— 适配接口, 通义/DeepSeek/Moonshot 可插拔
export interface LlmProvider {
  extract(messages: Msg[], schema: ZodSchema): Promise<ExtractResult>;  // 结构化抽取
}
// 通义/DeepSeek 均提供 OpenAI 兼容端点 → 用 @langchain/openai ChatOpenAI + 自定义 baseURL
// 结构化输出: withStructuredOutput(zodSchema) 或 JSON mode + Zod 二次校验兜底
```
- **省钱三件套（7.4）**：① 语义缓存——同 State 槽位组合命中直接返已生成卡片 JSON，绕过 LLM；
  ② 规则字典 Redis 预热——引擎查表走毫秒级缓存而非每次读 PG JSONB；③ 限流排队——LLM 触配额时降级"加速通道"排队。
- 日志脱敏（7.3）：`messages` 落库前对连续 8 位以上数字串（疑似电话/HKID）正则掩码为 `***`。

## 部署与 NFR（需求书第 7 章）

**Phase 1（DEMO/POC）**：单机部署，前端 Flutter Web 静态资源(CDN/Nginx) + NestJS 同机；LangGraph 用
`MemorySaver`(SQLite) 或单机 PG 维持上下文；支持 20–50 并发不崩即可。
延迟界限：LLM 反问 P90 `<5s`（配骨架屏）；规则引擎纯计算 P95 `<1s`（秒级渲染卡片）。

**Phase 2（生产，约 200 CCU）**：
- Node 天然异步，无 ASGI 阻塞问题；多核用 **PM2 cluster** 或容器多副本 + LB（替代 Gunicorn worker）。
- PostgreSQL 配 **PgBouncer** 连接池（防 200 并发打爆，LangGraph 高频读写 State）。
- 前端 CDN/Nginx；NestJS Docker 镜像 + CI/CD 滚动发布（零停机）；最外层 ALB 分发 HTTP/WebSocket。
- **低成本弹性（7.4，千元/月预算）**：Serverless 容器（阿里云 FC / 微信云托管，按量计费）+ 虚拟排队削峰 + 语义缓存绕过 LLM + 规则字典内存预热。

## 验证方式（端到端）

1. **契约单测**：对 `packages/contracts` 每个 Zod schema 写 `safeParse` 正/反例（含未知 `type` 降级）。
2. **基准剧本回放（需求书 2.6，作为验收用例）**：
   - **剧本一 Happy Path**：快捷胶囊→反问产品→年期→身份→出卡片→NLP 静默微调 PI/Non-PI 重算。断言每轮只反问 1 个槽位、卡片 5 分区齐全。
   - **剧本二 Edge Case**：Bright Universal Life RMB + Native Input 输 350000 + PI → 断言强制亮红色 Claw Back 退佣 banner + 橙色 SMPA 继续率提示。
3. **引擎单测**：永明四步公式对拍（边界：保费 0 / 9999999、N 超 eligibleYears、PI 仅首年）；断言不出负数。
4. **沙箱测试**：喂死循环/越界/`os` 调用代码，断言 3s 超时 Kill、128MB 限制、断网断 IO 生效。
5. **导入管道**：传"20%"未转小数、`Yr 1-200` 越界的 Excel，断言熔断并返回具体行列号。
6. **会话恢复**：中途断开重连，断言从 checkpoint 恢复到中断的反问节点。
7. **本地起服务**：`docker-compose up` 起 PG/Redis → `pnpm --filter @insurance/api dev` + `cd apps/app && flutter run -d chrome`，
   浏览器移动端模拟器走完两套剧本；Admin 端验证 HITL 审批 + 一键回滚。
